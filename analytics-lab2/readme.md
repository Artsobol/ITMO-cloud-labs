## Ход работы

1. Чтобы выполнить данную работу, нам понадобилось изучить структуру таблицы, по которой мы затем уже классифицировали потребление сервисов. Таблица довольно схожа с таблицей из лабораторной работы 1. Ниже представлена таблица, описывающая иерархию классификации облачных сервисов в данной модели:

| **Название столбца**   | **Описание**                                                                                                                                         | **Примеры**                                                 |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------- |
| **IT Tower**           | Основная категория услуг, описывающая крупные области облачных технологий и их функционал.                                                           | Compute, Storage, Database                                  |
| **Service Family**     | Группа услуг, выполняющая определённые функции и относящаяся к конкретной задаче по инфраструктуре.                                                  | Management & Governance, Application Integration, Analytics |
| **Service Type**       | Конкретный тип сервиса внутри группы сервисов. Отражает его основное предназначение                                                                  | Microsoft Virtual Machines, Microsoft Azure Database        |
| **Service Sub Type**   | Уточнение функционала конкретного типа сервисов. Показывает, какие специфические возможности или аспекты сервиса используются                        | Azure Database, Azure Backup                                |
| **Service Usage Type** | Тип использования услуги, который указывает, каким образом и для каких целей потребляется ресурс. Учитывает подтипы и назначение сервиса             | Tax, Cloud Deployment, Windows Cooperation.                 |
| **Meter Category**     | Указывает на тип ресурса или услуги, предоставляемой Azure                                                                                           | Analysis Services, Azure Databricks                         |
| **Meter Sub-Category** | Указывает какой именно ресурс или услуга используется в рамках основной категории. Она позволяет более детально отслеживать использование и расходы. | Tabular, Basic                                              |
| **Meter Name**         | Название конкретного счетчика, который измеряет использование ресурса или услуги Azure                                                               | Standard S%, Basic B%                                       |
| **Consumed Service**   | Определяет, к какому сервису относится использование, чтобы связать затраты с конкретным продуктом или платформой Azure                              | Microsoft.AnalysisServices                                  |

2. Мы сопоставили исходные данные с документацией Azure, при этом сохранив логическую структуру из лабораторной работы 1 и выделили следующие сервисы:

| **Название сервиса**              | **Описание**                                                                                                                                                                                                                                                                                          | **Примеры использования**                                                    |
| --------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| **Microsoft Azure Data Factory**  | Облачный сервис для перемещения и преобразования данных между разными системами в облаке | Автоматическое копирование данных из локальной БД в облачное хранилище для последующего анализа
| **Microsoft Azure Database**      | Сервис для работы с базами данных, таких как Azure SQL Database, Azure Database for MySQL и PostgreSQL, а также NoSQL базы данных, например, Cosmos DB                                                                                                                                                | Хранение и управление данными для веб-приложений                             |
| **Microsoft Azure Databricks**    | Облачная платформа для анализа данных и ML, которая объединяет возможности Apache Spark с простотой использования Azure для работы с big data                            | Анализ больших объемов данных о продажах для выявления трендов и создания рекомендаций |
| **Microsoft Azure Redis Cache**   | Облачный кэш-сервис, основанный на Redis, ускоряющий приложения за счет хранения часто используемых данных в памяти                                     | Хранение сессий пользователей, чтобы ускорить загрузку веб-приложения и снизить нагрузку на БД |
| **Microsoft Azure CDN**    | Облачная сеть доставки контента, ускоряющая загрузку веб-ресурсов, размещая их копии на серверах ближе к пользователям     | Кэширование статических файлов веб-приложения, чтобы снизить нагрузку на сервер |                                                     
| **Microsoft Azure Container Instances**     | Сервис для быстрой и простой развертки контейнеров в облаке без необходимости управления ВМ или инфраструктурой | Развертывание веб-приложения в контейнере, чтобы протестировать новую версию без настройки сервера или ВМ                                      |
| **Microsoft Azure Machine Learning**       | Облачная платформа для создания, обучения, развертывания и управления моделями ML             | Разработка и обучение модели машинного обучения для предсказания оттока клиентов, а затем ее развертывание в виде веб-службы для интеграции с CRM-системой |
| **Microsoft Azure Search**          | Облачный сервис поиска, позволяющий добавлять индексацию, фильтрацию, автодополнение и др. мощные возможности                                         | Создание поиска по каталогу интернет-магазина с функцией автоподсказок и фильтрации по цене или бренду                                                             |
| **Microsoft Azure VPN Gateway**     | Сервис для обеспечения защищенного подключения между локальной сетью и сетью Azure через VPN                                             | Безопасное подключение офиса компании к ВМ в Azure для работы сотрудников через защищенный канал |
| **Microsoft Azure Traffic Manager**      | Облачный DNS-регулятор, распределяющий входящий трафик между разными серверами или регионами для повышения доступности и производительности приложений  | Перенаправление пользователей на ближайший к ним сервер веб-приложения, чтобы снизить задержки |
| **Microsoft Azure Firewall**  | Облачный сетевой брендмауэр, обеспечивает защиту ресурсов в Azure, включая управление трафиком, фильтрацию URL и предотварщение угроз   | Блокировка несанкционированного доступа к БД, разрешая трафик только из определенных IP-адресов |
| **Microsoft Azure Notification Hubs**        | Облачный сервис для быстрой и масштабируемой отправки push-уведомлений на мобильные устройства   | Расслыка уведомлений о новых сообщениях пользователям мобильного приложения мессенджера |
| **Microsoft Power BI** | Платформа бизнес-аналитики для визуализации данных, создания интерактивных отчетов и панелей мониторинга для принятия решений | Построение дашборда, показывающего выручку, прибыль, региональные тренды с данными из облачной БД |
| **Microsoft Power BI Embedded**       | Сервис, позволяющий встраивать отчеты и дашборды Power BI в сторонние приложения или веб-сайты | Интеграция аналитической панели в корпоративное приложение |
